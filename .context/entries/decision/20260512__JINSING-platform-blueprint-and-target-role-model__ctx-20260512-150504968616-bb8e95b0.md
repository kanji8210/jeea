## What
Added a project blueprint document that defines platform layers, target modules, the nine-role operating model, capability design rules, core data domains, MVP boundaries, delivery phases, and the next implementation slices.

## Why
The project is shifting from isolated feature work to a full construction operations platform and needed a single standard for scope, permissions, module boundaries, and implementation order.

## Impact
Future work can now align around a shared architecture, especially for role expansion, unified finance, project-scoped permissions, document workflows, and phased product delivery.

## Notes
Explicitly deferred external accounting integrations for now and reserved Kenya compliance as a future capability bundle or dedicated role.
